randstr - A simple tool for generating data.

Welcome!

I have written randstr with the intention of allowing users to generate random data with a "format string-esque" style straight from the terminal. In this directory, you will find a few files generated purely from randstr as an example, with the format string placed in the file name.

Usage:

If you simply run randstr without any arguments, it prints the help prompt: 
Usage: randstr [OPTIONS]...

Random String Generator by Clara Van Nguyen.
Takes paramaters and generates a random string of data directed to stdout.

The following parameters are possible:
  -h	Shows this help prompt and exits
  -s	Specifies size (ex. -s=23)
  -g	Specifies generation seed, must be an integer (ex. -g=1234)
  -r	Specifies repeats/number of strings created (ex. -r=2)
  -l	Adds lower case letters to generation
  -u	Adds upper case letters to generation
  -n	Adds numbers to generation
  -a	Adds all possible characters to generation

It should be noted that -l, -u, -n, and -a can be added multiple times. Doing
so will increase the chances of those values getting hit by the random number
generator, as it will re-add them into the library of possible characters.
So now what? Why don't we generate some data?

Examples:

1. Single line of capital letters

Let's start off with generating a single line of 32 capital letters. That requires the -s=32 and -u parameters. 
UNIX> randstr -s=32 -u
UITWGUCKMNFACPMUXPSPFPELYKLPQUGK

2. Single line of capital and lowercase letters

The first example put letters A-Z into the pool to be generated. If we add in -l then we will also throw in a-z into the pool. Let's give it a shot. 
UNIX> randstr -s=32 -l -u
zECjymqQErqtgiQsmOgrSgAqAzIKKMzL

3. Single line of capital and lowercase letters... and numbers?

As you'd expect, adding in -n will add numbers into the mix. 
UNIX> randstr -s=32 -l -u -n
Xc4PbuQEicnYksRITwMRhXqrMYDhlkz6

4. Make numbers more likely to be generated

You can add parameters multiple times, and it'll make them more likely to appear. Let's make numbers four times more likely to show up. 
UNIX> randstr -s=32 -l -u -n -n -n -n
54011686e4R78ezbE59DZ27e6q076bG1

6. "Shorthand" Format String

Specifying the parameters in the way we've been doing it gets kind of tiresome. So there is a way to throw everything into a single string.

You can do the following:
  • Omit the "=" in the -s and -r parameters
  • Put multiple parameters in a single "-"
Let me show you what I mean. Example 3 in shorthand would be -s32lun: 
UNIX> randstr -s32lun
Xc4PbuQEicnYksRITwMRhXqrMYDhlkz6
Example 4 in shorthand would be -s32lunnnn: 
UNIX> randstr -s32lunnnn
54011686e4R78ezbE59DZ27e6q076bG1
Example 5 in shorthand would be -s32r16lun: 
UNIX> randstr -s32r16lun

7. Binary data

You may add in the -a parameter to add in all possible characters into the pool. This includes A-Z, a-z, 0-9, and every other character between 0 and 255.
